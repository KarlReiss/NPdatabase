# 前端开发计划（基于现有文档与原型）

**更新日期**: 2026-01-29  
**适用范围**: V1.1 前端实现（Vue 3）  
**依据文档**:  
- `docs/requirements-simplified.md`（页面结构、功能范围、UI/UX 规范）  
- `docs/next-steps.md`（前端联调优先级）  
- `frontend/prototype-react/README.md`（React 原型页面结构与迁移建议）

---

## 目标与交付物

**目标**: 以 React 原型为参照，先完成整体风格评审与改造建议，再用 Vue 3 完整实现核心页面，最后做细节打磨与优化。  

**主要交付物**:
- Vue 3 前端项目（页面结构与路由齐全）
- 关键页面：Home / List / Detail（与原型功能对齐）
- 与后端 API 的基础联调（搜索、列表、详情）
- 视觉风格与组件规范（基于用户提出的整体风格修改意见）

---

## 阶段计划

### Phase 0：准备与材料确认
**目标**: 明确参考与范围，为风格评审做准备。  
**动作**:
- 确认参考文件与路径：
  - React 原型：`frontend/prototype-react/`
  - 需求文档：`docs/requirements-simplified.md`
  - 联调计划：`docs/next-steps.md`
- 明确一期范围：仅实现核心浏览、列表、详情与搜索，不做二期扩展。

**产出**: 确认版的“参考清单 + 一期范围边界”。

---

### Phase 1：打开并评审 React 原型（**必须先完成**）
**目标**: 先看现有 React 原型页面，形成整体风格修改意见。  
**动作**:
- 逐页查看原型页面结构与视觉风格：
  - `App.tsx`（布局与路由）
  - `pages/Home.tsx`（首页）
  - `pages/ListPage.tsx`（列表页）
  - `pages/DetailPage.tsx`（详情页）
  - `components/Header.tsx`（导航头部）
- 记录当前风格特征（主色、字体、卡片/表格/筛选器风格）。
- 输出“整体风格修改意见”，形成统一的视觉方向与组件规范（由你提出）。

**产出**:
- 风格修改意见清单（颜色/排版/组件/信息层级）
- 风格基线（确定是否沿用 `#10B981` 或升级为新配色体系）

---

### Phase 2：Vue 3 架构搭建与页面复刻（结构先行）
**目标**: 完成 Vue 项目骨架与页面结构复刻，确保功能路径贯通。  
**动作**:
- 创建 Vue 3 项目结构与目录（含路由与基础布局）。
- 按原型页面结构映射到 Vue 组件：
  - `Home.vue` ← `pages/Home.tsx`
  - `CompoundList.vue` ← `pages/ListPage.tsx`
  - `CompoundDetail.vue` ← `pages/DetailPage.tsx`
- 先完成“结构与占位”实现，不进行细节美化。
- 预留 API 接入点（搜索 / 列表 / 详情）。

**产出**:
- 可运行的 Vue 3 前端框架
- 页面间路由与主布局可用

---

### Phase 3：基于风格意见进行全局样式实现
**目标**: 把“整体风格修改意见”落地为全局样式和组件规范。  
**动作**:
- 制定全局样式变量（颜色、字体、间距、阴影、圆角）。
- 替换原型视觉样式，形成一致的 UI 语言。
- 统一表格、筛选器、标签页、卡片的视觉规范。

**产出**:
- 统一主题样式与基础组件视觉规范

---

### Phase 4：API 联调与核心功能对接
**目标**: 与后端接口打通，完成主流程“搜索 → 列表 → 详情”。  
**动作**:
- 对接 `/api/search`、`/api/natural-products`、`/api/natural-products/:npId`。
- 依据需求文档实现筛选参数映射（MW/XLogP/PSA/活性等）。
- 实现分页与排序，确保结果可用与稳定。

**产出**:
- 核心流程可完成真实数据浏览

---

### Phase 5：细节修改与体验优化（**最后阶段**）
**目标**: 细节调整、体验优化、边界处理。  
**动作**:
- 空值/缺失字段展示策略优化（避免界面破碎感）。
- 表格/标签页细节（计数显示、空态、loading、错误态）。
- 响应式适配与基础可访问性优化。

**产出**:
- 细节完成版本，可进入验收

---

## 里程碑与验收标准

- **M1（风格完成）**: 风格修改意见明确并落实到全局样式  
- **M2（结构完成）**: Vue 页面结构与路由可用  
- **M3（联调完成）**: 搜索 → 列表 → 详情可用真实数据  
- **M4（细节完成）**: 体验细节完整、基本无明显空态问题

---

## 依赖与注意事项

- 后端 API 已完成，但仍需确认接口字段与分页参数的最终形式。  
- 若风格改动较大，应先冻结风格方向，再进入细节组件开发。  
- 详情页标签（Clinical Trials）一期可保留占位，不实现功能。
